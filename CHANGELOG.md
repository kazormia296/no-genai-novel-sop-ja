# 改訂履歴

この文書は、規範的変更と非規範的変更を区別して記録する。

## [1.2.0] - 2026-07-17

### 追加

- 基礎・追加プロファイルを保証軸ごとに比較する非単調な保証軸マトリクス。
- `SELF-ATTESTED / SECOND-PARTY / INDEPENDENT` の監査保証区分と外部固定証拠台帳。
- `PASS / CONDITIONAL PASS / FAIL` の規範的判定、条件付き表示、是正期限および再監査要件。
- TSV列、必須値、日時、列挙値、ID、外部キー、訂正方法を定める機械可読スキーマと生成文書。
- P4-Hのカットオフ、技術、根拠、構成物、証拠対応、ビルド環境、ブートストラップ種および例外リンクを分離した正規化台帳。
- スキーマ同期・値・参照整合性検証、Markdownリンク検査、厳密なリリース識別子検査、単体テストおよびGitHub Actions。

### 変更

- SOP本文と全配布テンプレートの列名を単一スキーマへ統一。
- リリース検証を厳密なSemVer、未追跡ファイル、チェックサム対象集合、注釈付きタグおよび文書間版一致の検査へ拡張。
- 版1.1.0の28列P4-H台帳を移行専用とし、新規適合記録では正規化台帳を必須化。

### 対象外

- 評価指摘3のP1とP2の中間プロファイルは追加していない。
- 評価指摘8の保守・承認・公開コメント等のリポジトリガバナンス体制は変更していない。

### 来歴

修正対象はリポジトリ所有者が明示的に指定し、規範文、スキーマ、検証実装および版管理上の反映にはOpenAI Codexを使用した。正式運用には人間の責任者による精査と承認を要する。

## [1.1.0] - 2026-07-17

### 追加

- P4を継承し、指定日時以後のソフトウェア供給網混入を排除するP4-H「歴史的カットオフ環境」プロファイル。
- カットオフ日時、根拠、構成物、保存証拠、ビルド環境、ブートストラップ種およびファームウェア境界を記録する専用台帳。
- 原時点バイナリ方式H-BINARYと歴史的ソース再構築方式H-SOURCEの適格性基準。
- 排除対象技術の利用可能時点より前に構成物が確定していた証拠により、対象技術に限って作成者表明を代替する経路。
- P4-H用の事故判定、最終監査項目および限定表示文。

### 変更

- 適合記録、最終監査票、README、版管理規則および公開手順をP4-Hと版1.1.0に対応。

### 来歴

P4-Hの導入方針はリポジトリ所有者が明示的に指示し、規範文、様式および版管理上の反映にはOpenAI Codexを使用した。正式運用には人間の責任者による精査と承認を要する。
